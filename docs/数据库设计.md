# 数据库相关说明

## 用户相关表

- **t_user（用户表）**

1. user_id（主键）
2. user_name
3. password
4. user_type（用户类型：普通用户/托运公司/管理员）
5. user_address
6. picture_id（头像）
7. company_id（如果是托运公司用户）

- **t_company（托运公司表）**

1. company_id（主键）
2. company_intro（公司介绍）
3. company_local（公司地址）

## 宠物相关表

- **t_pet（宠物表）**

1. pet_id（主键）
2. user_id（外键，关联用户表）
3. pet_name
4. pet_breed（品种）
5. pet_weight（体重）
6. pet_health_status（健康状况）

## 订单相关表

- **t_order（订单表）**

1. order_id（主键）
2. pet_id（外键，关联宠物表）
3. user_id（外键，关联用户表）
4. company_id（外键，关联公司表）
5. order_status（订单状态）
6. pet_status（宠物状态）
7. delivery_status（运输状态）
8. start_time（开始时间）
9. end_time（结束时间）
10. start_location（起始地点）
11. end_location（目的地）

## 图片相关表

- **t_picture（图片表）**

1. picture_id（主键）
2. picture_usage（图片用途）
3. picture_url（图片URL）
4. upload_time（上传时间）
5. user_id（上传用户）

## 社交相关表

- **t_post（帖子表）**

1. post_id（主键）
2. user_id（外键，关联用户表）
3. post_content（帖子内容）
4. post_comment（评论数）
5. create_time（创建时间）
6. t_comment（评论表）
7. comment_id（主键）
8. company_id（外键，关联公司表）
9. user_id（外键，关联用户表）
10. comment_content（评论内容）
11. create_time（创建时间）

## 系统相关表

- **t_system_log（系统日志表）**

1. log_id（主键）
2. user_id（操作用户）
3. operation_type（操作类型）
4. operation_content（操作内容）
5. operation_time（操作时间）

## 表关系说明：

- 一个用户可以拥有多个宠物（一对多）
- 一个用户可以创建多个订单（一对多）
- 一个托运公司可以处理多个订单（一对多）
- 一个用户可以发布多个帖子（一对多）
- 一个用户可以发表多个评论（一对多）
- 一个托运公司可以收到多个评论（一对多）

## 建议的字段类型：

- ID字段：使用VARCHAR(18)或UUID

- 状态字段：使用CHAR(1)或TINYINT

- 时间字段：使用DATETIME
- 文本字段：根据长度使用VARCHAR或TEXT
- 数值字段：根据范围使用INT或DECIMAL

## 索引设计：

- 主键索引：所有表的主键
- 外键索引：所有外键字段
- 常用查询字段：如user_name, order_status等
- 时间字段：如create_time, operation_time等
